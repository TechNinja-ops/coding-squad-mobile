Recap: 

What is GitHub

GitHub is where your code is uploaded to such that it can be accessed on the internet. GitHub also serves other functions such as collaborative work and static web hosting. We would be using both these features in the course of using this platform. To begin go ahead and create a GitHub account here.

Git on the other hand is a product of GitHub. Git used used for version control which ideally is a method developers use to monitor changes made to their code. GIT comes in two ways, as a CLI (Command Line Interface) and GUI (Graphical User Interface). For the purpose of this lesson we would be interacting with git using the CLI.

GIT CLI is a set of written commands used to interact with GitHub. To begin, you need to download the GIT CLI from their website. After doing this you should download the Upload Guide written by an instructor of Coding Squad, Onyekwelu Ifunanya. The material is available on the code library however it was made for a particular product of Coding Squad but it provides a good explanation of using GIT on both a laptop and mobile device (android).

Note that if you need help with your code from us, you need to upload your code to GitHub and add us as collaborators before we can assist you.

Understanding GitHub

You know how folders are on your PC and how you can store your files in them? That's how there are folders on GitHub, only that instead of folders, we call them repositories!

You can have your repositories as either private or public depending on what you're trying to do. With a private repository only you and whoever you allow can see your code base which enhances security. The downside of this is that you will not be able to use the feature of GitHub that allows you to host your websites unless you upgrade to a paid plan.

With a public repository, anyone with or without a GitHub account can see your code and "fork" (copy to their own profile as a repository) but to make direct changes to your code, you would have to add the user as a collaborator or they would have to perform an action called a pull request. We wouldn't go deep into it here, to further get the insides of GitHub you need to be in the Git and Firebase course!

Creating a GitHub Repository

There are two broad categories a website can fall under. A Static or Dynamic website. A static website is a site that remains the same and the content for all users remain the same. A dynamic website is quite the opposite, a site where the content can vary with the user.

The Mobirise software only generates static websites and the platforms discussed here for hosting your website both host only static websites. GitHub Pages is a service provided by GitHub where you can host your static websites completely free. You may use this to show your client the website before adding a paid domain.

When you get to the next lesson on Firebase usage, you'll understand that we can add some dynamic properties to our static websites such as making dashboards for users. This is still considered as static as HTML, CSS and JavaScript still remain the technologies used.
