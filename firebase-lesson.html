What is Firebase

In the world of programming, there are two major stacks you can venture, either a frontend or backend developer or a combination of both (Full stack developer). Learning frontend means you can build the things that users can actually use and interact with. But there's a whole root below which you may not have knowledge on.

The backend is often called the dynamic part of a website. Things like user authentication and state management, API and routes, Database management and Server management happens back there and coding these from scratch can be very tedious.

As you know, this program only equips you with frontend knowledge. You may at some point need some backend functions and that's where Firebase comes in. Firebase is a developer kit that allows developers to make static websites work dynamically using backend functions. Firebase has user authentication, database, storage facilities, crash analytics, machine learning and best of all hosting!

You can host static websites like the sites you use the Mobirise software to build and you can host it completely free! Although using other features aside hosting will require some coding, we got you covered on that. So far you upload your project.mobirise file on GitHub we can assist with your projects!

Firebase web hosting also gives you a unique free domain (.web.app). However if you host a mobile app like the coding Squad mobile app, you also get a ".app" domain although you pay for it (less than 500NGN). The only way to host your website on Firebase is using Firebase CLI. If you've not before, go ahead and install something called NodeJS. When you install node you will automatically get something called Node Package Manager or npm for short.

There are two ways to actually host your website on Firebase. Either you do so directly or you use a GitHub Automation. If you use a direct connection, you would always need a terminal to update your website's content as the Firebase CLI is only available over a CLI. With a GitHub Automation, you can simply send the code to your repository and GitHub takes care of the rest with Firebase. We shall show you both ways but there's something we need to clarify

To use the Firebase CLI, you need to install the Firebase CLI tools to your PC, to install the Firebase CLI to your PC you need node, to use node you need a PC. But in the event you don't have a PC, you can still use a terminal environment on your ANDROID mobile device. But it occupies your phone, it doesn't harm the phone at all just makes your phone a little full. Ask questions in the discussions page if you need the step on for your phone.
