Post Development

We've come a long way in the no code web development course. There are certain things that you should have to do after you have created the website on Mobirise. The next thing would obviously be to publish the website to your preferred location, we would use local storage as a case study.

When you deploy your website to your local storage, the next thing is to modify certain parts of the code. There is a section in every webpage you design with the Mobirise Software where there is a tag that says things like "Made with Mobirise". It's not very professional having such a thing on your client's page.

Remember in the HTML bit of the course, we discussed that there are semantic elements which add no specific change to the website other than showing the different blocks you have on the website. This tag is hardcoded in the page and to remove it, this is where we would need VS code

This branding tag is usually the last section between the footer section and the script tags. To begin open VS code extensions and download the beautify extension, we would explain why we need this in a bit.

Installing the beautify VS code extension.

The next step is to open the project folder on VS Code and then opening the beautify extension on the code. This will arrange all the lines of code in a better formatted environment according to their class names and sections. You would scroll down to before you have script tags and you should see a section with a class name of "display-7" go ahead and delete the entire section.

Best Practice For Continuous Development

If you would maybe be in constant need of updating the content of the website you should do the following things.
  1. Dev Folder: Within the main folder for the project, create another folder and name it Dev.
  2. Prod Folder. In the same location as the Dev folder, make a folder named Prod.
  3. Publish: When publishing to local storage, set the location of your files to always be in Dev. This means the code and assets will be stored in the Dev folder.
  4. Merge: For the first time you publish to the dev folder, copy the entire content of the folder to the Prod folder and make the necessary modifications such as adding git for version control, removing the branding, adding more texts or text formatting etc.
  5. Deploy: Ensure that if you wish to publish your code to git, you use the Prod folder which would act as your local production server.
  6. Updates: If you update the site from Mobirise such as adding a new page, publish to the Dev folder then copy only the assets folder and the additional pages to the Prod. If you made changes to an existing file in the Prod, copy the changes in text from the Dev to the Prod for the particular file or replace altogether if the changes are minimal.
