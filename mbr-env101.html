Technologies used in Mobirise sites

Bootstrap CSS & JS 

Remember the different screen views settings we discussed? Mobirise is able to help you create sites that are fully adapted to different screen sizes using Bootstrap. Bootstrap creates sites that are responsive so you don't need to worry about the site's responsiveness. Bootstrap CSS creates the layout for all the various while Bootstrap JS identifies the screen size and applies the CSS styles accordingly.

Animate CSS

Animate CSS is an animation library that allows you to apply animation to different sections on your website. Anytime you drag any block element into the canvas, that is regarded as a section. By default there is just one animation class which is top to bottom. You may change the classes for each element to apply a different style. Note that the animations only work views above the tablet horizontal view.

Vimeo Player JS

Vimeo Player JS file is a library that allows developers to embed and control Vimeo videos on your website. To add a video to your site, the video should be uploaded on YouTube or Vimeo so as to allow quick and easy playback, looping and auto play

Masonry Layout JS

Masonry JS file is a library that allows for the creation of dynamic grid layouts called masonry layouts. It makes arranging images and texts blocks look flexible and responsive. The Masonry grid is only available in the start M5 theme. You may find it under Gallery and Sliders as well as the features element.

Jarallax (CSS & JS)

Jarallax is a CSS and Javascript library that is used to add a parallax effect on pictures across your website. Whenever you use the parallax toggle, you activate this library on that element

Smoothscroll JS

Smooth scroll is a Javascript Library that allows fluid and easy vertical scrolling on images

Formoid JS

Formoid JS is a static API designed by Mobirise which allows you to have forms get sent directly to your email. This API is only available on pages that have forms.

Images Loaded 

If you want us to build backend functions for you to create certain triggers on images, we can do that using thie JS library. Images Loaded JS library is generally used to detect when an image has loaded and trigger certain events on a Dynamic Server

Theme (CSS & JS)

The theme file contains a CSS and a JS file which is essentially the front end theme. There are different themes available on Mobirise. As you get those themes the CSS and JS libraries of themes gets updated. 

Embla (CSS & JS)

Embla is the library that's responsible for those scrolling carosels you see, the animation library for auto scroll and button control and visibility are all objects in the embla library

Dropdown (CSS & JS)

You may add dropdown menu to your menu bar. Adding a dropdown menu allows you to add more menu options to a particular menu option. Simply hover on a menu item and you'll see the dropdown icon whcich you can use to add dropdown menu

Scroll Gallery JS

Very similar smooth scroll but this library only allows for horizontal automatic scroll animation for images.

Web Assets (fonts and icons)

All the icons you see when you click the sun icon when you hover an image, we design them ourselves and when you export your project, you export this with the project.

Mobirise files (CSS & JS) 

In addition to the theme's code in the themes folder, we usually add more CSS and Js code usually based on modifcations you make along the way in your project. They are additional CSS and JS files for the theme.

Project.mobirise

The project.mobirise file is a file that contains the full working directory of your full Mobirise project. For us to collaborate with you on your website, you'll need to also upload the file along side all the assets and HTML codes.

Advanced Block Editing

Here are some key things to modify in each element you drag on the canvas
Padding

Padding is defined as the external top and bottom spacing between elements in a page. When you hover around a block you would see the settings icon. When you click it, you would see two options for top and bottom and moving the scale will increase or decrease the space between the blocks above or below the target block.

Colors and Opacity

Still on the settings of each block, you have the option of changing the background color and setting it to an image and even a video. When you set an image you may see an option to parallax the image, which activates the Jarallax script. Also you may set an overlay color and set it's opacity. If you choose a video you should use a link.

Block content

Some blocks have internal cards which you can customize it's color. Look for the option for show/hide which toggles usually the top part of the block's content. Then the card section where other parts of the card can be toggled or modifed

Publish

After all the necessary changes has been done to your code, the next thing is to publish. There are about 4 ways of publishing your code and we have discussed each of them below

Preview

The preview function in Mobirise allows you to see the look of a page in a browser. The difference between preview and publish is that the former will only store the source code in a temporary folder which gets deleted when the browser is closed.

GDPR Complaince

Under the site settings, the General Data Protection Regulation is an option you can toggle to enable this on your site. This basically informs users that by filling a form on your site, they are consenting to you processing their information as such when you activate this, it's only present on forms. 

Cookies

In web development, a cookie is basically information that your device sends to a web server. The major aim why we added this option is if you wish to have a backend job on your site. You would contact us and we would add this to your site. You may have your policy to show on the center of your user's screen or at the bottom when page loads.

Analytics, Robots and Sitemap

If you wish to know how many users visit your site on a daily basis, we can build an analytics script for you. Robots file is a text file that prevents web crawling bot agents to use your site to collect data and a Sitemap is used to show all the different pages of a website diagrammatically

WEBP Conversion

Just like png and jpeg, WebP is another picture format that has rapidly become popular and compatible with many browsers. WebP is known to reduce server latency while still maintaining optimal performance. For this reason we have an option of converting images to WebP. It's advisable to do this only after you're done adding images.

Text editing and links

You can select a text or a group of texts and make them bold, italicized or add a link to them. When you want to add a link to a text, after highlighting the text, the options for bold, italic and link will appear. For link you have the option to link an existing page on your web project and choose a particular section you want users to navigate to, use an external website, link to a phone number or mail, link to a file such as a downloadable pdf, or link to a file on your main server like a python, JS or PHP file. Same goes if you're linking a button on a component

Pixabay Images
Pixabay is a free open source API that allows developers to get clean, restriction free images for their projects. To use Pixabay, anytime you want to add a picture rather than selecting browse computer or Site library use Free Online Library. As the name implies, you're gonna have to be online When using Pixabay to source for images for your projects, note the following.

  • Rate Limits: Rate limit in API development is how many times a service will respond to a query before the latency (gap for reply) is maxed. In Pixabay you can generate images only after a 1 minute break between queries.
  • Query form: A query is something that is searched for. In Pixabay, it's bad practice to use a sentence to search for an image. Pixabay isn't AI or Machine learning. If you search for something like apple, you may see fruits and see an IPhone!
  • Caching: Pixabay will automatically save your last query and use this to populate the search home page. This means when you search for something and close the picture dialogue box and reopen it, you will see items related to your last search
