Exploring CSS

The ultimate goal of CSS is to add style to your web page, how does this happen though? How does CSS know what element to apply styles to? The answer is something called Semantic Elements. Now let's say your mom tells you to "...Get me that my dress", assuming there are a lot of dresses that belong to her, what's the natural response you'd give her? "...Which one?", right? but let's assume each dress has a different design and she says "...Give me that one that has a cat picture", if there's just one dress that has a cat picture, that's the dress you'd give. That's kind of how CSS works.

The picture above is a line from a HTML file. In HTML the div tag (<div></div>) means division and it's main function is to group similar HTML elements. But if you notice that line in the picture above says <div class="container">. What does class="container" mean? Class in CSS is called a Selector and it works just like the dress example given.

In the dress example, the instruction was to bring a cat-designed dress. In simpler terms out of all the dresses, bring a cat-designed dress. In HTML there WILL be a lot of "divs" for different parts of the web page for this reason there has to be a way for CSS to be able to apply styles to a specific div without necessarily affecting all other divs.

The implication of this is that any text, video or image inside a div element that has this class with the name "container", we can modify it in CSS. A "div" isn't the only thing you may find in a HTML page for grouping similar elements. Another prominent one is called "section".

Note that both "div" and "section" have closing tags! If you notice for section there's another thing called "id". Now an "id" is a little different from a "class" though they are both selectors in CSS which means you can apply styles to both of them on a "div" and "section".

In our software, when you generate the website's source code, you'll notice that only the sections have "id" and "class" while divs have just "class". This is because a div or a section can accommodate multiple "classes" but just one "id". What does this even mean? In CSS we can have a rule for a class that will change the color, font type and size for all text and give it a class name of "text" this means that anywhere there is the class name of text, the texts in the div or section will have the set color, font-type and size. We can still add the "container" class name inside that same section and give it completely different styles. 

Sounds confusing right? In the picture above, the class names are gallery3 and cid-uqkkhHEmE0. These class names have different rules that they apply to the items in that section but the id name is just gallery03-1g. On this particular page if there is another id with this value it would not function at all and if on that line there was another id name like how there's two class names, both ids will not function.

Another thing is don't go about in the CSS script and make spelling adjustments, CSS was designed in American English so it's "color" and not "colour". While we may not discuss all the CSS rules, if you encounter a rule and you'd like to know more about it, you can always just ask in the discussions page.
